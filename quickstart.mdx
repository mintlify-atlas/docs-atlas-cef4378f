---
title: "Get started with scans2any"
description: "Convert your first infrastructure scan in minutes. Download a sample Nmap file, run a conversion, and explore combining multiple scans."
---

This guide walks you through your first scans2any conversions using a real Nmap sample file. By the end you will know how to convert a single scan, change the output format, combine multiple scans, and auto-detect scans in a directory.

<Steps>
  <Step title="Install scans2any">
    Install scans2any using `uv`, `pipx`, or Docker. The fastest path is:

    ```bash
    uv venv
    uv pip install .
    source .venv/bin/activate
    ```

    See the [installation guide](/installation) for all options including Docker and `uv run` (no install required).
  </Step>

  <Step title="Download a sample scan">
    Grab the official Nmap sample XML file used throughout this guide:

    ```bash
    wget https://raw.githubusercontent.com/nmap/nmap/refs/heads/master/zenmap/radialnet/share/sample/nmap_example.xml
    ```
  </Step>

  <Step title="Run your first conversion">
    Pass the file to scans2any with `--nmap`. The default output format is a formatted terminal table:

    ```bash
    scans2any --nmap nmap_example.xml
    ```

    You will see a `fancy_grid` table printed to your terminal, with columns for IP addresses, hostnames, ports, services, banners, and OS:

    ```text
    ╒════════════════╤══════════════════╤══════════╤════════════╤════════════════════════════════╤═══════╕
    │ IP-Addresses   │ Hostnames        │ Ports    │ Services   │ Banners                        │ OS    │
    ╞════════════════╪══════════════════╪══════════╪════════════╪════════════════════════════════╪═══════╡
    │ 188.68.47.54   │ a2f36.netcup.net │ 21/tcp   │ ftp        │ ProFTPD                        │ linux │
    │                │ softscheck.com   │ 22/tcp   │ ssh        │ OpenSSH/8.4p1 Debian 5+deb11u4 │       │
    │                │                  │ 53/tcp   │ domain     │                                │       │
    │                │                  │ 80/tcp   │ http       │ nginx                          │       │
    │                │                  │ 443/tcp  │ http       │ nginx                          │       │
    │                │                  │ 8443/tcp │ https-alt  │ sw-cp-server                   │       │
    ╘════════════════╧══════════════════╧══════════╧════════════╧════════════════════════════════╧═══════╛
    ```
  </Step>

  <Step title="Change the output format">
    Use `-w` to select a different writer. To output JSON:

    ```bash
    scans2any --nmap nmap_example.xml -w json
    ```

    ```json
    {
      "188.68.47.54": {
        "hostnames": [
          "a2f36.netcup.net",
          "softscheck.com"
        ],
        "tcp_ports": {
          "21": { "service_names": ["ftp"], "banners": ["ProFTPD"] },
          "22": { "service_names": ["ssh"], "banners": ["OpenSSH/8.4p1 Debian 5+deb11u4"] },
          "53": { "service_names": ["domain"], "banners": [""] },
          "80": { "service_names": ["http"], "banners": ["nginx"] },
          "443": { "service_names": ["http"], "banners": ["nginx"] },
          "8443": { "service_names": ["https-alt"], "banners": ["sw-cp-server"] }
        },
        "udp_ports": {},
        "os": ["linux"]
      }
    }
    ```

    Save the output to a file with `-o`:

    ```bash
    scans2any --nmap nmap_example.xml -w json -o result.json
    ```

    <Tip>
      Run `scans2any -W` to see all 15 available writers with their descriptions and options.
    </Tip>
  </Step>

  <Step title="Combine multiple scans">
    Pass multiple files of the same or different types. scans2any merges them into a single infrastructure:

    ```bash
    scans2any --nmap tcp_scan.xml --nmap udp_scan.xml --nessus vulnerabilities.nessus
    ```

    Hosts that appear in more than one scan are automatically deduplicated. If any conflicting data cannot be resolved automatically, scans2any writes a `MERGE_FILE.yaml` and tells you how to proceed.

    <Tip>
      See [Merging scans](/usage/merging-scans) for a full walkthrough of conflict resolution and merge files.
    </Tip>
  </Step>

  <Step title="Auto-detect scans in a directory">
    Use `--auto` (or `-a`) to let scans2any detect and import all supported scan files in a directory:

    ```bash
    scans2any --auto tests/data
    ```

    This is useful when you have a folder of mixed scan types and do not want to specify each file individually.
  </Step>
</Steps>
