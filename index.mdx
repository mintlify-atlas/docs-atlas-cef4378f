scans2any is a command-line tool that aggregates infrastructure scan results from multiple sources — Nmap, Nessus, Masscan, Aquatone, Bloodhound, and NetExec — and converts them into whatever output format you need for reporting, further analysis, or tool chaining. It handles conflicting scan data intelligently, letting you resolve collisions automatically or manually.

## How it works

scans2any processes scans in three phases: parse and combine, merge and filter, then check and output.

<Steps>
  <Step title="Parse and combine">
    Feed any combination of scan files — Nmap XML, Nessus `.nessus`, Masscan JSON, and more. scans2any parses each file and combines them into a single unified infrastructure using Union-Find clustering to identify the same host across multiple scans.
  </Step>
  <Step title="Merge and filter">
    Conflicts — like two scans reporting different service names for the same port — are resolved automatically using built-in rules. Apply filters to normalize banners, remove noise, and focus on relevant hosts and services.
  </Step>
  <Step title="Output in any format">
    Export to terminal tables, Markdown, HTML, CSV, JSON, YAML, XML, LaTeX, Typst, Excel, or generate Nmap re-scan scripts and Aquatone URL lists. Choose a single output or chain formats together.
  </Step>
</Steps>
